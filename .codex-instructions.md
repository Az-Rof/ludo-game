# Ludo Game Development

## Your Task
Read PLAN.md for full specifications. You are implementing Phase 1: Core Board + Single Player.

## Phase 1 Requirements
- HTML structure with lobby (username input, create/join room) and game area
- CSS board: 15×15 grid, standard Ludo cross/plus shape
- Colors: Red, Blue, Green, Yellow (classic)
- Canvas rendering for tokens (simple circles) and dice (CSS dots)
- Dice roll: click to roll, random 1-6, brief animation
- Token selection: click token to move
- Step-by-step token movement animation
- Turn management (2-4 players, local)
- Capture logic (opponent token sent home)
- Home column logic (only accessible with exact roll)
- Win condition (all 4 tokens home)

## Technical Requirements
- Frontend only (no server yet - Phase 2)
- Vanilla JS, no frameworks
- All graphics via CSS and Canvas (no external assets)
- Responsive design (works on mobile)
- File structure:
  - public/index.html
  - public/css/style.css
  - public/css/board.css
  - public/js/main.js
  - public/js/board.js
  - public/js/dice.js
  - public/js/game.js
  - public/js/ui.js

## Rules
- Classic Ludo rules (standard worldwide)
- Double dice: roll again on 6
- 3 consecutive 6s: turn forfeited
- Safe squares: star squares
- Token movement: exact count required

## Quality
- Clean, well-structured code
- Comments where helpful
- Mobile-friendly
- Smooth animations
